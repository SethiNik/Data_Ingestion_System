Component Responsibilities

  • Frontend handles ingestion requests and previews data.
  • Backend extracts tables and infers schema.
  • Kafka decouples ingestion and persistence.
  • Consumer persists records into MySQL.
  • Dashboard enables ingestion validation.

Scalability Model

  • Multiple producers can ingest concurrently.
  • Kafka partitions allow parallel consumers.
  • Database writes can be batched.
  • Services can scale independently.

Production Deployment Model

  • Containerized services via Docker.
  • Environment driven configuration.
  • Stateless ingestion services.
  • Reusable ingestion pipelines.
